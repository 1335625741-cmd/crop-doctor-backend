# Feedback Collection Flow(反馈收集流程)

> 诊断 + 用药推荐已给出后,启动反馈收集。流程最多 3 轮,以"满意度评分"收尾。

## 何时启动

**不要**在 skill 主流程结束时立即收集反馈。
**要**把"反馈邀请"作为回复的最后一段自然语言,等用户主动回答后再展开下一轮。

```
诊断回复(判断 + 严重程度 + 现在能做的 + 推荐用药 + 要不要找人)
  ↓
最后一段:反馈邀请(等用户回答)
```

---

## Round 1 — 初始反馈

**问题模板**(用下面的措辞,4 个选项,不要重新措辞):

> 按上面的建议处理后,问题有没有改善?
>
> **A. 解决了** — 问题已消除
> **B. 改善一些** — 部分好转
> **C. 没变化** — 还那样
> **D. 恶化了** — 更严重了
> **E. 还没处理** — 我打算试试看 / 等天晴再处理

**根据用户回答分流**:

| 回答 | 行动 |
|---|---|
| **A** | 进入 Round 3 "满意度收集",跳到收尾 |
| **B** | 简短致谢 + 鼓励(有些病本来就要 2-3 次复喷,好转说明方向对了) + 询问要不要补充建议;如果用户说"够了",进入 Round 3 |
| **C / D** | 进入 Round 2 "深入询问" |
| **E** | 不收反馈(还没处理没法评估);回答"好的,处理 5-7 天后记得回来告诉我效果,我好根据情况调整建议";结束本轮(等用户再次来找) |

---

## Round 2 — 深入询问(只在 C/D 分支触发)

**目的**:弄清"为什么没改善/恶化了",然后给出针对性调整方案。

**步骤 1: 询问具体情况**(4 个问题,一次性问完):

> "我想帮你调整一下方案,先了解几个情况(可多选/单选):"
>
> **Q1. 具体做了什么?**
> - A. 按推荐用药喷了
> - B. 只摘了病叶,没打药
> - C. 自己换了别的药
> - D. 还没做任何处理
> - E. 其他(请说)
>
> **Q2. 如果用了药,剂量对得上吗?**
> - A. 完全按推荐剂量
> - B. 差不多但不完全一致
> - C. 不清楚怎么兑水
>
> **Q3. 喷完后下雨了吗?**
> - A. 没下雨
> - B. 喷后 6 小时内下了
> - C. 不确定
>
> **Q4. 病斑扩散情况?**
> - A. 只在原位,没扩大
> - B. 略扩大但不多
> - C. 大幅扩散
> - D. 别的部位也开始出现

**步骤 2: 根据回答调整方案**(用以下决策表):

| 关键信号 | 调整方案 |
|---|---|
| Q1=A, Q2=A, Q3=B | **雨水冲掉了** → 重新喷 + 加"防雨小窍门"(关注 24h 天气预报,喷药选晴天) |
| Q1=A, Q2=C | **剂量不对** → 详细解释兑水方法(给具体数字步骤,如"先把 10 g 药加少量水搅成糊状,再兑到 15 kg 水里") |
| Q1=C | **换了药可能冲突** → 警告"不同药剂不要随意混用或换用,先停药 5-7 天再用推荐方案" |
| Q1=D | **还没处理** → 回到 Round 1 的 E 分支,提醒处理 |
| Q4=C / Q4=D | **情况恶化** → 升级严重度,直接建议**当天联系当地农技员**;不再给方案 |
| 多种信号 | 综合判断,文字说明哪条最关键,按关键路径调整 |

**步骤 3: 给出调整后的方案,然后回到 Round 1**:

> "按上面的调整处理 5-7 天后再看看。这次有什么需要补充的吗?"

→ 重新走 Round 1 的 A/B/C/D/E 分支。

**最大轮数**:Round 2 最多触发 **2 次**。如果 3 轮都没解决,直接转 Round 3 收尾,不再继续 drill-down。

---

## Round 3 — 满意度收集(最终轮)

无论诊断是否解决问题,这一轮都触发,目的是收集**量化反馈**用于改进。

**问题模板**:

> 最后想请你打个分,方便我改进:
>
> **整体满意度**(1-5 星,直接回数字即可)
>
> ⭐ 1 — 完全没用
> ⭐⭐ 2 — 一般,没解决但有些参考
> ⭐⭐⭐ 3 — 还行,有点帮助
> ⭐⭐⭐⭐ 4 — 不错,基本解决问题
> ⭐⭐⭐⭐⭐ 5 — 非常满意,完全解决

**如果满意度 ≤ 3,继续问不满意原因**(多选):

> 不太满意的话,主要原因是什么?(可多选)
> **A. 诊断结果不对** — 不是这个病
> **B. 推荐药买不到** — 当地没货
> **C. 方案太复杂** — 看不懂怎么做
> **D. 用药后没效果**
> **E. 病情恶化了**
> **F. 其他**(请说)

**如果满意度 ≥ 4,简短致谢 + 询问满意原因**(可选):

> 很高兴帮到你!方便说一下哪些地方对你帮助最大吗?
> A. 病名判断准
> B. 用药具体(包括商品名和剂量)
> C. 处置动作明确
> D. 给了本地农资店/农技员建议
> E. 其他

**收尾话术**:

- 满意度 ≥ 4: "感谢反馈!需要其他农业问题随时找我 🌾"
- 满意度 ≤ 3: "谢谢你的反馈,我会基于这些改进方案。下次有需要再来 🌱"

## 反馈记录格式

收集到的反馈用 `bin/log_feedback.py` 写入 JSONL 文件,每条反馈一个 JSON 对象:

```json
{
  "session_id": "producer 任务 ID 或时间戳",
  "skill_version": "v6",
  "diagnosis": {
    "name": "黄瓜白粉病",
    "category": "病害",
    "confidence": "高"
  },
  "rounds": [
    {"round": 1, "user_response": "D"},
    {"round": 2, "drill_down": {"Q1": "A", "Q2": "B", "Q3": "B", "Q4": "C"}}
  ],
  "final_satisfaction": 3,
  "dissatisfaction_reasons": ["用药后没效果"],
  "satisfaction_reasons": [],
  "free_text": "喷了 2 次没好转,病斑还在扩大",
  "collected_at": "2026-07-05T18:30:00+08:00"
}
```

**字段说明**:

- `session_id`: 关联 producer 任务的唯一 ID(可选,后期用于 join)
- `diagnosis`: 当时的诊断快照(便于后续分析"哪种病最容易差评")
- `rounds`: 多轮交互记录(1-3 轮)
- `final_satisfaction`: 1-5 整数
- `dissatisfaction_reasons`: 不满意原因列表,对应 Q 的选项 A-F
- `satisfaction_reasons`: 满意原因列表,可选
- `free_text`: 自由文本,农户可能补充额外信息
- `collected_at`: ISO 8601 时间戳

---

## 反馈存放位置

**默认**:`~/.mavis/skills/crop-disease-diagnosis/feedback/feedback-log.jsonl`

每条反馈一行 JSON(JSONL 格式),便于后续:
- `cat feedback-log.jsonl | jq` —— 命令行查看
- 用 Python `pandas` 聚合统计(平均满意度、按病种拆分、不满意原因 Top N)
- 跑 `bin/log_feedback.py --stats` —— 内置统计(将来实现)

---

## 隐私与数据保留

- 反馈文件存在**本地**,不上传任何远程
- 收集的字段**不包含用户身份信息**(姓名、手机号、地址)
- 农户填的"作物种类 / 地点 / 气候"是诊断需要的上下文,不算隐私
- 农户不应在 free_text 里写个人敏感信息(生产者侧应在反馈收集页注明)

## 关联 bin/log_feedback.py

收完反馈后,producer 调 `bin/log_feedback.py` 把数据落到 JSONL:

```bash
# 命令行参数式
python bin/log_feedback.py \
  --session-id "task-2026-07-05-001" \
  --diagnosis-name "黄瓜白粉病" \
  --diagnosis-category "病害" \
  --satisfaction 4 \
  --positive-reason "用药具体" \
  --free-text "推荐很清楚,大生买到了"

# JSON stdin 式(更灵活,适合 producer 拼好完整 JSON 后传)
echo '{"diagnosis":{"name":"黄瓜白粉病"},"satisfaction":4,...}' | \
  python bin/log_feedback.py --stdin
```

详见 `bin/log_feedback.py --help`。
